Sobre o sistema

O Goodfunds é um sistema de controle financeiro pessoal desenvolvido para uso próprio. Cobre avaliação de faturas, planejamento financeiro, estimativas, dashboards e gestão de transações e categorias.

Linguagem
Java 17
Framework
Spring Boot 3
Banco (prod)
PostgreSQL
Frontend
React + Vite

Objetivos do MVP

  • Registrar e categorizar receitas e despesas manualmente.
  • Importar faturas de cartão (PDF Nubank) e processar transações automaticamente.
  • Planejar gastos mensais com orçamentos por categoria.
  • Visualizar relatórios, evolução e estimativas financeiras.

Segurança

Modelo de autenticação

  • JWT (JSON Web Token) com expiração de 24 horas.
  • Senhas armazenadas com BCrypt.
  • Filtro JwtAuthFilter valida o token antes de qualquer controller.
  • Todas as rotas exigem JWT, exceto /auth/**, /actuator/health, /actuator/info e /swagger-ui/**.
  • Cada usuário acessa somente seus próprios dados (verificação no service via SecurityContext).

Como executar

Pré-requisitos

  • Java 17+
  • PostgreSQL rodando em localhost:5432 com database goodfunds (perfil dev)
  • Bash ou terminal compatível com Maven Wrapper

Iniciar o backend

cd backend
./mvnw spring-boot:run

A aplicação sobe em http://localhost:8080 com o perfil dev.

Rodar os testes

cd backend
./mvnw verify

Utiliza H2 in-memory com o perfil test. Não requer PostgreSQL.

Health check

curl http://localhost:8080/actuator/health

Swagger UI

http://localhost:8080/swagger-ui.html

Decisões técnicas

DecisãoMotivo
open-in-view=falseEvita lazy loading fora da transação e problemas de performance
ddl-auto=none em dev/testFlyway gerencia o schema também em desenvolvimento e testes
ddl-auto=validate em prodFlyway é a única fonte verdade do schema em produção
PDFs no filesystemEvita inflar o banco com blobs binários
ProblemDetail (RFC 7807)Formato padronizado e interoperável de erros HTTP
Caffeine em /reports/*Relatórios são leitura intensiva; cache reduz carga no banco
JWT sem refresh tokenSimplificação intencional para o MVP
Interface InvoiceParserArquitetura preparada para parsers de outros bancos além do Nubank
Paginação em todos os endpoints de listagemEvita retornos ilimitados conforme os dados crescem
